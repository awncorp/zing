
=encoding utf8

=head1 NAME

Zing::KeyVal - Key/Value Store

=cut

=head1 ABSTRACT

Generic Key/Value Store

=cut

=head1 SYNOPSIS

  use Zing::KeyVal;

  my $keyval = Zing::KeyVal->new(name => 'notes');

  # $keyval->recv;

=cut

=head1 DESCRIPTION

This package provides a general-purpose key/value store abstraction.

=cut

=head1 INHERITS

This package inherits behaviors from:

L<Zing::Repo>

=cut

=head1 LIBRARIES

This package uses type constraints from:

L<Zing::Types>

=cut

=head1 ATTRIBUTES

This package has the following attributes:

=cut

=head2 name

  name(Str)

This attribute is read-only, accepts C<(Str)> values, and is optional.

=cut

=head1 METHODS

This package implements the following methods:

=cut

=head2 poll

  poll() : Poll

The poll method returns a L<Zing::Poll> object which can be used to perform a
blocking-fetch from the store.

=over 4

=item poll example #1

  # given: synopsis

  $keyval->poll;

=back

=cut

=head2 recv

  recv() : Maybe[HashRef]

The recv method fetches the data (if any) from the store.

=over 4

=item recv example #1

  # given: synopsis

  $keyval->recv;

=back

=over 4

=item recv example #2

  # given: synopsis

  $keyval->send({ status => 'happy' });

  $keyval->recv;

=back

=cut

=head2 send

  send(HashRef $value) : Str

The send method commits data to the store overwriting any existing data.

=over 4

=item send example #1

  # given: synopsis

  $keyval->send({ status => 'happy' });

=back

=over 4

=item send example #2

  # given: synopsis

  $keyval->drop;

  $keyval->send({ status => 'happy' });

=back

=cut

=head2 term

  term() : Str

The term method generates a term (safe string) for the datastore.

=over 4

=item term example #1

  # given: synopsis

  $keyval->term;

=back

=cut
